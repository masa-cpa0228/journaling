## ローカルでの確認方法

### おすすめ：VS Code拡張機能「Live Preview」を使う

更新後の見た目をすぐ確認したい場合は、Microsoft公式の VS Code 拡張機能 `Live Preview` を使う方法がシンプルです。

1. VS Codeでこのフォルダを開く。
2. 拡張機能の画面で `Live Preview` をインストールする。
   - 提供元が `Microsoft` のものを選んでください。
3. `index.html` を開く。
4. 画面右上のプレビューアイコン、または右クリックメニューから `Live Preview` を開く。

保存した変更がプレビューに反映されるため、ターミナルで毎回サーバーを起動する必要がありません。

### 予備：VS Codeのタスクから起動する

毎回ターミナルにコマンドを打たなくても、VS Codeのタスクから確認用サーバーを起動できます。

1. VS Codeでこのフォルダを開く。
2. メニューから `Terminal` → `Run Task...` を選ぶ。
3. `サイト確認用サーバーを起動` を選ぶ。
4. ブラウザで `http://localhost:3001/` を開く。

更新後はブラウザを再読み込みすると、最新の内容を確認できます。

### ターミナルから起動する場合

このフォルダで以下を実行：

```bash
python3 server.py 3001
```

ブラウザで `http://localhost:3001/` を開くとサイトが表示されます。
（`server.py` は Python 標準機能だけで動く簡易Webサーバーです。）

### スマートフォンで確認する場合

スマートフォンで確認する場合も、HTMLファイルを単体で開くより、Macで起動したローカルサーバーへアクセスする方法がおすすめです。
同じWi-Fiに接続したうえで、スマートフォンのブラウザから次の形式で開きます。

```text
http://MacのIPアドレス:3001
```

MacのIPアドレスは、ターミナルで次のコマンドを実行すると確認できます。

```bash
ipconfig getifaddr en0
```

もし上のコマンドで表示されない場合は、次のコマンドを実行し、`inet` の後ろに表示される `192.168...` のような数字を確認します。

```bash
ifconfig en0
```

HTMLファイルをスマートフォンへ送って直接開くこともできますが、CSSやページ間リンクを含めた確認がしづらくなるため、基本的にはローカルサーバー経由で確認します。

## 開発の進め方

- 詳しい開発計画とステップ分割は [PROGRESS.md](PROGRESS.md) を参照。
- **1ステップ ＝ 1ブランチ ＝ 1プルリクエスト** の単位で進めます。
- 各ステップ完了時に [PROGRESS.md](PROGRESS.md) を更新します。
- 「次に書きたい記事ネタ」「気になる改善点」は GitHub の Issue で別管理します。
